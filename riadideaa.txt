You are an expert fullstack developer and UX designer specializing in 
hospitality websites for boutique Moroccan properties.

Build a complete, production-quality riad guesthouse website using the 
MERN stack (MongoDB, Express, React + Vite, Node.js) with Tailwind CSS.

---

## CONTEXT

The client is a small riad or guesthouse in Asilah, Morocco — a coastal 
medina town known for its blue-and-white painted walls, murals, and 
Atlantic character. The property has 4–8 rooms, offers breakfast, and 
currently relies on Booking.com (losing 15–18% commission per booking).

The goal of the site is to:
1. Replace Booking.com as the primary booking channel
2. Build trust with international tourists (European, mostly French-speaking)
3. Convert visitors via WhatsApp as the primary CTA
4. Be manageable by a non-technical riad owner via a simple admin panel

## PAGES & SECTIONS

### Public site (React frontend)

1. **Hero** — Full-bleed image, property name in Playfair Display, 
   tagline in French/English, floating WhatsApp CTA button (sticky)

2. **About** — Warm narrative about the riad's history, the medina, 
   the owners. Split layout: text left, photo mosaic right.

3. **Rooms** — Card grid (3 per row, collapses to 1 on mobile). Each 
   card: room photo, name, bed type, capacity, short description, 
   "Book via WhatsApp" button that pre-fills a message.

4. **Gallery** — Masonry photo grid with lightbox (use yet-another-react-
   lightbox). Photos pulled from Cloudinary.

5. **Experience** — What guests can do: medina walk, beach, local 
   hammam, cooking class. Icon + short text cards.

6. **Testimonials** — 3-column review cards with guest name, country 
   flag emoji, rating stars, review text. Pulled from MongoDB.

7. **Booking / Contact** — Simple inquiry form: name, email, check-in, 
   check-out, number of guests, message. Sends email via Nodemailer + 
   stores inquiry in MongoDB. No payment processing needed.

8. **Footer** — Address, phone, WhatsApp, Google Maps link, language 
   switcher (AR / FR / EN using i18next)

### Admin panel (protected route, JWT auth)

- Login page
- Dashboard: list of booking inquiries (name, dates, guests, status)
- Ability to mark inquiry as: pending / confirmed / cancelled
- Room manager: add/edit/delete rooms with Cloudinary image upload
- Testimonials manager: add/edit/delete reviews
- Gallery manager: upload/delete photos

## BACKEND (Express + MongoDB)

### Models (Mongoose)
Room: { name, description, bedType, capacity, pricePerNight,
images: [cloudinaryUrl], amenities: [String], isAvailable }
Inquiry: { guestName, email, checkIn, checkOut, guests, message,
status: enum['pending','confirmed','cancelled'], createdAt }
Testimonial: { guestName, country, rating, text, isVisible }
GalleryImage: { url, caption, order }
Admin: { email, passwordHash }

### API routes
POST   /api/inquiries          — submit booking inquiry (public)
GET    /api/inquiries          — list all (admin only)
PATCH  /api/inquiries/:id      — update status (admin only)
GET    /api/rooms              — list all rooms (public)
POST   /api/rooms              — create room (admin only)
PUT    /api/rooms/:id          — update room (admin only)
DELETE /api/rooms/:id          — delete room (admin only)
GET    /api/testimonials       — list visible (public)
POST   /api/testimonials       — add (admin only)
DELETE /api/testimonials/:id   — delete (admin only)
GET    /api/gallery            — list all images (public)
POST   /api/gallery/upload     — upload to Cloudinary (admin only)
DELETE /api/gallery/:id        — delete (admin only)
POST   /api/auth/login         — returns JWT
GET    /api/auth/me            — verify token

## WHATSAPP INTEGRATION

Every "Book Now" button should open:
https://wa.me/212XXXXXXXXX?text=Bonjour%2C%20je%20souhaite%20r%C3%A9server...
Pre-fill the message with room name and dates if the user has interacted 
with the booking form. Make this a reusable `<WhatsAppButton>` component.

## MULTILINGUAL (i18next)

Support 3 languages: French (default), English, Arabic (RTL).
Use `react-i18next`. Store translations in `/locales/fr.json`, 
`/locales/en.json`, `/locales/ar.json`.
Language switcher in the navbar and footer.
On Arabic, apply `dir="rtl"` to the `<html>` tag dynamically.
